# LinkedIn Post - git-vanish

🔥 **I just published an open-source CLI tool that every developer needs — `git-vanish`**

We've all been there.

You accidentally commit a `.env` file, AWS credentials, or a private key to GitHub.
You panic. You delete the file. You push again.

**But the secret is still there. In every commit. Forever. Visible to anyone who clones your repo.**

The standard fix (`git filter-branch`) is painful, confusing, and easy to get wrong.
So I built something better.

---

**Introducing `git-vanish` ⚡**

A beautiful terminal UI that lets you:
✅ Browse your repo like a file explorer — fully keyboard-driven
✅ Select **multiple files** at once with `Space`
✅ Erase them from **every single commit** in your entire git history
✅ Keep the file **safe on your disk** — only removed from git, not from your machine
✅ Optionally add to `.gitignore` to prevent future accidents

All in one command:

```
npx git-vanish
```

---

No more copy-pasting scary `filter-branch` commands from Stack Overflow.
No more accidentally deleting your own files.
No more anxiety about what's buried in your old commits.

**Install it globally and never worry again:**

```
npm install -g git-vanish
```
